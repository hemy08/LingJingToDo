## 1. 快速开始

### 1.1 安装应用

#### Windows
1. 下载 `LingJingToDo_x.x.x_x64-setup.exe` 安装包
2. 双击运行安装程序
3. 按照安装向导完成安装
4. 启动应用

#### macOS
1. 下载 `LingJingToDo_x.x.x_x64.dmg` 安装包
2. 双击打开 DMG 文件
3. 将应用拖入 Applications 文件夹
4. 启动应用

#### Linux
1. 下载 `lingjingtodo_x.x.x_amd64.AppImage` 文件
2. 添加执行权限：`chmod +x lingjingtodo_x.x.x_amd64.AppImage`
3. 运行应用：`./lingjingtodo_x.x.x_amd64.AppImage`

### 1.2 首次使用

启动应用后，您将看到主界面：

1. **左侧**：日历面板，用于选择日期
2. **中间**：任务面板，显示当前日期的任务
3. **右侧**：统计面板，显示任务统计信息
4. **顶部**：自定义标题栏，包含窗口控制按钮

### 1.3 创建第一个任务

1. 在任务面板顶部找到任务添加区域
2. 输入任务标题（必填）
3. 选择任务状态、类型、优先级（可选）
4. 设置截止日期（可选）
5. 点击"添加"按钮或按 Enter 键

恭喜！您已经创建了第一个任务。

## 3. 任务管理

### 3.1 添加任务

#### 添加主任务
1. 在任务添加区域输入任务标题
2. （可选）选择状态、类型、优先级
3. （可选）设置截止日期
4. 点击"添加"按钮或按 Enter 键

#### 添加子任务
1. 找到要添加子任务的主任务
2. 点击主任务卡片上的"添加子任务"按钮
3. 在弹出的模态框中输入子任务信息
4. 点击"确定"按钮

### 3.2 编辑任务

#### 编辑主任务
1. 点击任务卡片进入编辑模式
2. 修改任务信息（标题、状态、类型、优先级、截止日期、备注）
3. 点击"保存"按钮或点击其他区域自动保存

#### 编辑子任务
1. 展开主任务查看子任务列表
2. 点击子任务进入编辑模式
3. 修改子任务信息
4. 点击"保存"按钮

### 3.3 删除任务

#### 删除主任务
1. 点击任务卡片上的"删除"按钮
2. 确认删除操作
3. 主任务及其所有子任务将被删除

#### 删除子任务
1. 展开主任务查看子任务列表
2. 点击子任务上的"删除"按钮
3. 确认删除操作

### 3.4 任务排序

#### 拖拽排序
1. 鼠标悬停在任务卡片上
2. 按住鼠标左键拖动任务卡片
3. 拖动到目标位置释放鼠标

#### 排序模式
- **插入模式**：拖动任务插入到目标位置
- **交换模式**：拖动任务与目标位置任务交换

### 3.5 任务筛选

#### 按状态筛选
1. 点击任务面板上的"筛选"按钮
2. 选择要筛选的状态
3. 只显示符合状态的任务

#### 按类型筛选
1. 点击任务面板上的"筛选"按钮
2. 选择要筛选的类型
3. 只显示符合类型的任务

#### 按优先级筛选
1. 点击任务面板上的"筛选"按钮
2. 选择要筛选的优先级
3. 只显示符合优先级的任务

### 3.6 任务搜索

1. 点击任务面板上的"搜索"按钮
2. 输入搜索关键词
3. 显示包含关键词的任务

## 9. 常见问题

### 9.1 安装问题

**Q: Windows 安装时提示"无法验证发布者"？**
A: 这是正常提示，点击"更多信息" → "仍要运行"即可。我们正在申请代码签名证书。

**Q: macOS 安装时提示"无法打开，因为它来自身份不明的开发者"？**
A: 右键点击应用，选择"打开"，然后在弹出的对话框中点击"打开"。

**Q: Linux 下无法运行 AppImage 文件？**
A: 确保已添加执行权限：`chmod +x lingjingtodo_x.x.x_amd64.AppImage`

### 9.2 使用问题

**Q: 任务数据保存在哪里？**
A: 数据保存在应用目录下的 `data/` 文件夹中：
- 任务数据：`data/tasks/YYYY-MM-DD.json`
- 配置数据：`data/config.json`

**Q: 如何备份任务数据？**
A: 复制整个 `data/` 文件夹到安全位置即可。或使用导出功能导出为 JSON/Excel/XML 文件。

**Q: 如何迁移到新电脑？**
A: 
1. 在旧电脑上导出任务数据
2. 在新电脑上安装应用
3. 导入任务数据

**Q: 任务太多，界面卡顿怎么办？**
A: 
- 使用筛选功能减少显示的任务数量
- 切换到列表视图提高性能
- 我们正在优化大量任务的渲染性能

**Q: 如何恢复误删的任务？**
A: 
- 目前没有回收站功能，删除操作不可撤销
- 建议定期备份数据
- 我们计划在未来版本添加回收站功能

### 9.3 文件操作问题

**Q: 导入 Excel 文件失败？**
A: 
- 确保文件格式为 `.xlsx` 或 `.xls`
- 确保文件包含必需的列（任务ID、标题等）
- 查看错误提示了解具体原因

**Q: 导出的 Excel 文件无法打开？**
A: 
- 确保使用 Microsoft Excel 或兼容软件打开
- 检查文件是否损坏
- 尝试重新导出

**Q: 导入数据会覆盖现有数据吗？**
A: 
- 导入操作会合并数据，不会覆盖
- 相同 ID 的任务会被更新
- 不同 ID 的任务会被添加

### 9.4 配置问题

**Q: 如何恢复默认配置？**
A: 
- 删除 `data/config.json` 文件
- 重启应用，将自动生成默认配置

**Q: 自定义的状态/类型/优先级删除后，相关任务会怎样？**
A: 
- 任务的状态/类型/优先级会重置为默认值
- 建议在删除前检查是否有任务使用该配置

## 10. 技巧与建议

### 10.1 任务管理技巧

#### 使用优先级
- **P0 (致命)**：必须立即处理的关键问题
- **P1 (严重)**：严重影响使用的问题
- **P2 (重要)**：重要的功能或任务
- **P3 (一般)**：常规任务
- **P4 (较低)**：优先级较低的任务
- **P5 (低)**：可以延后的任务
- **P6 (不紧急)**：有空再处理的任务

#### 使用状态
- **待规划**：刚创建，还未开始规划
- **已启动**：已开始规划，但未开始执行
- **进行中**：正在执行中
- **已完成**：已完成所有工作
- **已延期**：超过截止日期但未完成
- **已关闭**：不再追踪的任务

#### 使用类型
- **工作**：工作相关的任务
- **学习**：学习相关的任务
- **生活**：生活相关的任务
- 可自定义添加更多类型

### 10.2 组织任务建议

#### 使用子任务
- 将大任务分解为多个子任务
- 每个子任务应该是可独立完成的
- 完成子任务后更新主任务状态

#### 设置截止日期
- 为重要任务设置截止日期
- 定期检查今日到期和已延期的任务
- 及时调整任务优先级

#### 定期回顾
- 每天查看今日到期任务
- 每周回顾所有任务
- 每月整理和归档已完成任务

### 10.3 数据管理建议

#### 定期备份
- 每周导出一次完整数据
- 保存多个版本的备份
- 使用云存储保存备份文件

#### 使用导出功能
- 导出为 Excel：方便手动编辑和查看
