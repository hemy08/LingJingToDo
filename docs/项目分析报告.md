## 3. 核心功能模块

### 3.1 任务管理模块

#### 3.1.1 数据模型

```typescript
interface Task {
  id: string                    // 任务唯一标识
  title: string                 // 任务标题
  status_id: string             // 状态ID (关联状态配置)
  type_id: string               // 类型ID (关联类型配置)
  priority_id: string           // 优先级ID (关联优先级配置)
  due_date?: string             // 截止日期 (YYYY-MM-DD)
  subtasks?: Task[]             // 子任务列表
  remark?: string               // 备注信息
  created_date?: string         // 创建日期
  closed_date?: string          // 关闭日期
}
```

#### 3.1.2 任务 ID 生成规则

**主任务 ID**：
- 格式：`TASK001`, `TASK002`, `TASK003`, ...
- 生成逻辑：查询当前日期所有任务，找到最大编号 + 1

**子任务 ID**：
- 格式：`SUB001`, `SUB002`, `SUB003`, ...
- 生成逻辑：查询父任务的所有子任务，找到最大编号 + 1

#### 3.1.3 任务操作流程

**添加任务流程**：
```
用户输入 → 前端验证 → 生成任务ID → 
调用 add_task API → Rust 后端处理 → 
保存到文件 → 返回结果 → 前端更新UI
```

**更新任务流程**：
```
用户编辑 → 前端验证 → 调用 update_task API → 
Rust 后端处理 → 更新文件 → 返回结果 → 
前端更新UI
```

**删除任务流程**：
```
用户确认 → 调用 delete_task API → 
Rust 后端处理 → 从文件删除 → 返回结果 → 
前端更新UI
```

fn schedule_auto_save() {
    let now = chrono::Local::now();
    let next_1am = now.date_naive().succ_opt().unwrap()
        .and_time(chrono::NaiveTime::from_hms_opt(1, 0, 0).unwrap());
    let duration = next_1am.signed_duration_since(now);
    
    thread::spawn(move || {
        thread::sleep(duration.to_std().unwrap());

    });
}

**后端状态**：
```rust
// 线程安全状态
pub struct TaskState {
    tasks: Mutex<HashMap<String, Vec<Task>>>,
}

// 状态更新
impl TaskState {
    pub fn update_tasks(&self, date: &str, tasks: Vec<Task>) {
        let mut map = self.tasks.lock().unwrap();
        map.insert(date.to_string(), tasks);
    }
}
```

### 4.3 数据持久化策略

**按日期分文件**：
- 优点：避免单文件过大，提高读写性能
- 缺点：跨日期查询需要读取多个文件

**配置独立存储**：
- 优点：配置与任务数据分离，便于管理
- 缺点：需要维护多个文件

**后端错误处理**：
```rust
pub fn get_tasks(date: String) -> Result<Vec<Task>, String> {
    let file_path = format!("data/tasks/{}.json", date);
    match fs::read_to_string(&file_path) {
        Ok(content) => {
            let tasks: Vec<Task> = serde_json::from_str(&content)
                .map_err(|e| format!("Failed to parse JSON: {}", e))?;
            Ok(tasks)
        }
        Err(e) => {
            // 文件不存在时返回空数组
            if e.kind() == std::io::ErrorKind::NotFound {
                Ok(vec![])
            } else {
                Err(format!("Failed to read file: {}", e))
            }
        }
    }
}
```
